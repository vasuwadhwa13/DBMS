Data Types

Data types define the nature of data that can be stored in each column of a table. Below are commonly used SQL data types:

Data Type Description Column Length
CHAR(n)Fixed length character data of length nFixed at n bytes with trailing spaces
VARCHAR(n)Variable length character dataUp to 2000 bytes per row
NUMBER(p,s)Numeric data with precision p and scale sUp to 21 bytes
DATEStores date and time dataFixed at 7 bytes
LONGVariable length character data up to 2GBUp to 2GB
RAW(n)Variable length raw binary dataUp to 2000 bytes

Example:

CREATE TABLE Students (
    StudentID NUMBER(5),
    Name VARCHAR(100),
    DOB DATE,
    Grade CHAR(2),
    Photo RAW(2000)
);

Creating Tables

The CREATE TABLE statement is used to create a new table in a database.

Syntax:

CREATE TABLE table_name (
    column1 datatype,
    column2 datatype,
    column3 datatype,
    ....
);

The column parameters specify the names of the columns of the table. The datatype parameter specifies the type of data the column can hold (e.g. varchar, integer, date, etc.).

Example:

CREATE TABLE Persons (
    PersonID int,
    LastName varchar(255),
    FirstName varchar(255),
    Address varchar(255),
    City varchar(255)
);

The empty "Persons" table can now be filled with data using the SQL INSERT INTO statement.

Create Table Using Another Table

You can also create a table using another table's structure and data.

Syntax:

CREATE TABLE new_table_name AS
SELECT column1, column2, ...
FROM existing_table_name
WHERE ...;

Example:

CREATE TABLE TestTable AS
SELECT customername, contactname
FROM customers;

Inserting Values

The INSERT INTO statement is used to insert new records into a table.

INSERT INTO Syntax

You can use the INSERT INTO statement in two ways:

  1. Specifying both column names and values: 
    INSERT INTO table_name (column1, column2, column3, ...)
    VALUES (value1, value2, value3, ...);
  2. Inserting values into all columns:

    Make sure the order of values matches the column order in the table.

    INSERT INTO table_name
    VALUES (value1, value2, value3, ...);

Updating Columns

The UPDATE statement is used to modify existing records in a table.

Syntax:

UPDATE table_name
SET column1 = value1, column2 = value2, ...
WHERE condition;

Important: The WHERE clause specifies which records should be updated. Without it, all records will be affected!

Example:

UPDATE Students
SET Grade = 'A+'
WHERE StudentID = 1001;

Deleting Rows

The DELETE statement is used to remove existing records from a table.

Syntax:

DELETE FROM table_name
WHERE condition;

Important: Be cautious! If you omit the WHERE clause, all records in the table will be deleted.

Example:

DELETE FROM Students
WHERE StudentID = 1001;

Dropping Column

The ALTER TABLE statement is used to delete a column from an existing table.

Syntax:

ALTER TABLE table_name
DROP COLUMN column_name;

Example:

ALTER TABLE Students
DROP COLUMN MiddleName;

Note: Some database systems may have limitations or alternative syntax for dropping columns.

Querying Multiple Tables

6.1 Collating Information

Collating information means collecting or gathering data from different tables. SQL enables you to build relationships between multiple tables and extract meaningful data using a single command.

6.1.1 SQL Joins

  • Equi Join
    Syntax: 
    SELECT * FROM table1, table2 WHERE table1.id = table2.id;
    Example: 
    SELECT employees.name, departments.name FROM employees, departments WHERE employees.dept_id = departments.id;
  • Cartesian Join
    Syntax: 
    SELECT * FROM table1, table2;
    Example: 
    SELECT * FROM students, courses;
    Returns every combination of student and course (not usually practical without a WHERE clause).
  • Outer Join
    Syntax: 
    SELECT * FROM table1 LEFT OUTER JOIN table2 ON table1.id = table2.id;
    Example: 
    SELECT employees.name, departments.name FROM employees LEFT JOIN departments ON employees.dept_id = departments.id;
  • Self Join
    Syntax: 
    SELECT A.name, B.name FROM employees A, employees B WHERE A.manager_id = B.id;
    Example: 
    SELECT E1.name AS Employee, E2.name AS Manager FROM employees E1 JOIN employees E2 ON E1.manager_id = E2.id;

6.1.2 Set Operators

  • UNION
    Syntax: 
    SELECT column_name FROM table1 UNION SELECT column_name FROM table2;
    Example: 
    SELECT city FROM customers UNION SELECT city FROM suppliers;
  • UNION ALL
    Syntax: 
    SELECT column_name FROM table1 UNION ALL SELECT column_name FROM table2;
    Example: 
    SELECT city FROM customers UNION ALL SELECT city FROM suppliers;
  • INTERSECT
    Syntax: 
    SELECT column_name FROM table1 INTERSECT SELECT column_name FROM table2;
    Example: 
    SELECT product_id FROM warehouse1 INTERSECT SELECT product_id FROM warehouse2;
  • MINUS
    Syntax: 
    SELECT column_name FROM table1 MINUS SELECT column_name FROM table2;
    Example: 
    SELECT employee_id FROM hr MINUS SELECT employee_id FROM payroll;

6.2 Nested Queries

6.2.1 Correlated Sub Query

A correlated subquery uses values from the outer query. It executes once for each row in the outer query.

Syntax: 
SELECT name FROM employees e WHERE salary > (SELECT AVG(salary) FROM employees WHERE dept_id = e.dept_id);
Example: 
SELECT name FROM students s WHERE marks > (SELECT AVG(marks) FROM students WHERE class = s.class);

6.2.2 Special Operators in Sub Queries

  • EXISTS
    Syntax: 
    SELECT name FROM employees WHERE EXISTS (SELECT * FROM departments WHERE employees.dept_id = departments.id);
  • ANY
    Syntax: 
    SELECT name FROM employees WHERE salary > ANY (SELECT salary FROM managers);
  • SOME
    Syntax: 
    SELECT name FROM employees WHERE salary > SOME (SELECT salary FROM interns);
    'SOME' is interchangeable with 'ANY'.
  • ALL
    Syntax: 
    SELECT name FROM employees WHERE salary > ALL (SELECT salary FROM contractors);

Conditional Retrieval

Conditional retrieval allows you to filter records based on specified conditions using the WHERE clause. SQL provides several conditional operators to match values precisely, find ranges, check for missing values, and perform pattern matching.

Overview of Using Other Conditional Operators

SQL supports various conditional operators such as =, >, <, BETWEEN, IN, LIKE, IS NULL, etc., for filtering data according to different needs.

Using the IN Operator

The IN operator allows checking if a value exists within a list of values. 


        -- Retrieve students in departments 1, 2, or 3
        SELECT * FROM Students
        WHERE DepartmentID IN (1, 2, 3);

Using the BETWEEN-AND Operator

The BETWEEN ... AND ... operator is used to retrieve values within a specific range (inclusive of boundary values). 


        -- Retrieve students with age between 18 and 25
        SELECT * FROM Students
        WHERE Age BETWEEN 18 AND 25;

Using the LIKE Operator

The LIKE operator allows pattern matching using wildcards. % represents any sequence of characters, and _ represents a single character. 


        -- Retrieve students whose name starts with 'A'
        SELECT * FROM Students
        WHERE Name LIKE 'A%';
        
        -- Retrieve students whose name ends with 'n'
        SELECT * FROM Students
        WHERE Name LIKE '%n';
        
        -- Retrieve students with a 5-letter name starting with 'R'
        SELECT * FROM Students
        WHERE Name LIKE 'R____';

Using a WHERE Clause

The WHERE clause in SQL is used to filter records that meet a specific condition. It is often used in SELECT, UPDATE, DELETE, and INSERT statements to narrow down the results based on one or more conditions.

Syntax

SELECT column1, column2, ...
              FROM table_name
              WHERE condition;

The condition is usually a comparison using operators like =, >, <, >=, <=, <> (not equal), or logical operators like AND, OR, and NOT.

Example: Filtering by Exact Match

SELECT * 
              FROM CUSTOMERS 
              WHERE AGE = 25;

This query selects all customers who are 25 years old.

Example: Using Multiple Conditions

SELECT * 
              FROM CUSTOMERS 
              WHERE AGE > 25 AND SALARY > 3000;

This returns customers whose age is greater than 25 and salary is greater than 3000.

Example: Using OR and NOT

SELECT * 
              FROM CUSTOMERS 
              WHERE AGE < 23 OR NOT (SALARY > 5000);

This fetches customers who are younger than 23 or who earn 5000 or less.

WHERE Clause with Strings

SELECT * 
              FROM CUSTOMERS 
              WHERE NAME = 'Ramesh';

This retrieves the customer whose name is exactly Ramesh.

WHERE Clause with Pattern Matching

You can also combine WHERE with the LIKE keyword for partial matches (covered in more detail in the Pattern Matching section):

SELECT * 
              FROM CUSTOMERS 
              WHERE NAME LIKE 'K%';

This query finds all customers whose names start with the letter "K".

Working with NULL

SQL uses the term NULL to represent a non-existent or unknown data value in the database. These values are not the same as an empty string or a zero. They don't hold any space in the database and are used to signify the absence of a value or the unknown value in a data field.

Some common reasons why a value may be NULL:

  • The value may not be provided during the data entry.
  • The value is not yet known.

Since NULL values are essentially non-existent, you cannot use comparison operators such as =, <, or > with them. However, you can check for NULL using the IS NULL or IS NOT NULL operators.

Creating a Table without NULL Values

NULL values can be inserted in any column of a table as they are not associated with any specific data type. However, when a column is defined with the NOT NULL keyword, an error is raised whenever you try to insert NULL values into that specific column.

Syntax: 
CREATE TABLE table_name (
            column1 datatype NOT NULL,
            column2 datatype NOT NULL,
            ...
            columnN datatype
          );
Example: 
CREATE TABLE CUSTOMERS (
            ID INT NOT NULL,
            NAME VARCHAR(20) NOT NULL,
            AGE INT NOT NULL,
            ADDRESS CHAR(25),
            SALARY DECIMAL(18, 2),
            PRIMARY KEY (ID)
          );

Inserting Records (Including NULL Values)

INSERT INTO CUSTOMERS VALUES 
          (1, 'Ramesh', 32, 'Ahmedabad', 2000.00),
          (2, 'Khilan', 25, 'Delhi', 1500.00),
          (3, 'Kaushik', 23, 'Kota', 2000.00),
          (4, 'Chaitali', 25, 'Mumbai', 6500.00),
          (5, 'Hardik', 27, 'Bhopal', 8500.00),
          (6, 'Komal', 22, 'Hyderabad', NULL),
          (7, 'Muffy', 24, 'Indore', NULL);

Querying Non-NULL Values

SELECT ID, NAME, AGE, ADDRESS, SALARY 
          FROM CUSTOMERS 
          WHERE SALARY IS NOT NULL;

Querying NULL Values

SELECT ID, NAME, AGE, ADDRESS, SALARY 
          FROM CUSTOMERS 
          WHERE SALARY IS NULL;

Updating NULL Values

UPDATE CUSTOMERS 
          SET SALARY = 9000 
          WHERE SALARY IS NULL;

Verifying the Update

SELECT * FROM CUSTOMERS;

Deleting Records with NULL Values

DELETE FROM CUSTOMERS 
          WHERE SALARY IS NULL;

Verifying the Deletion

SELECT * FROM CUSTOMERS;

SQL Pattern Matching with LIKE

The LIKE operator is used in a WHERE clause to search for a specified pattern in a column. It is commonly used with wildcards to filter results based on partial matches.

Wildcards used with LIKE:

  • % — Represents zero, one, or multiple characters
  • _ — Represents a single character

Syntax:

SELECT column1, column2, ...
          FROM table_name
          WHERE columnN LIKE pattern;

SQL ORDER BY Clause

The ORDER BY keyword is used to sort the result set in either ascending or descending order based on one or more columns. By default, the ORDER BY clause sorts the data in ascending (ASC) order.

Syntax:

SELECT column1, column2, ...
          FROM table_name
          ORDER BY column1 [ASC|DESC], column2 [ASC|DESC], ...;

          4         | Chef Anton's Cajun Seasoning   | 2          | 2          | 48 - 6 oz jars         | 22

Sort by Numeric Column (Ascending)

SELECT * FROM PRODUCTS

Sort by Numeric Column (Descending)

SELECT * FROM PRODUCTS
          ORDER BY Price DESC;

SELECT * FROM PRODUCTS
          ORDER BY ProductName;

SELECT * FROM PRODUCTS

This query sorts all customers by their Country. If multiple customers are from the same country, they are further sorted by CustomerName.

This query sorts customers in ascending order of Country and descending order of CustomerName within each country. 

SELECT * FROM CUSTOMERS
          ORDER BY Country ASC, CustomerName DESC;

SQL Aggregate Functions

An aggregate function performs a calculation on a set of values and returns a single value. These functions are especially useful for summarizing data and are often used in combination with the GROUP BY clause.

The most commonly used aggregate functions in SQL are:

  • MIN() – returns the smallest value in a column
  • MAX() – returns the largest value in a column
  • COUNT() – returns the number of rows that match a condition
  • SUM() – returns the total sum of a numeric column
  • AVG() – returns the average value of a numeric column

Note: All aggregate functions ignore NULL values except COUNT().

Syntax:

SELECT AGGREGATE_FUNCTION(column_name)
          FROM table_name
          [WHERE condition];

Sample Data

We will use the ORDERS table for the examples below:

OrderID | CustomerID | Amount
          -------------------------------
          1       | C001       | 2500
          2       | C002       | 3600
          3       | C001       | 1900
          4       | C003       | 4100
          5       | C002       | NULL
          6       | C004       | 1500
          7       | C001       | 2900

Using MIN()

SELECT MIN(Amount) AS MinimumAmount
          FROM ORDERS;

Using MAX()

SELECT MAX(Amount) AS MaximumAmount
          FROM ORDERS;

Using COUNT()

Returns the number of non-NULL entries in the Amount column:

SELECT COUNT(Amount) AS TotalTransactions
          FROM ORDERS;

Returns the total number of rows, including those with NULL values:

SELECT COUNT(*) AS TotalOrders
          FROM ORDERS;

Using SUM()

SELECT SUM(Amount) AS TotalSales
          FROM ORDERS;

Using AVG()

SELECT AVG(Amount) AS AverageOrder
          FROM ORDERS;

Using Aggregate Functions with WHERE Clause

SELECT COUNT(*) AS HighValueOrders
          FROM ORDERS
          WHERE Amount > 2500;

Using Aggregate Functions with GROUP BY

SELECT CustomerID, SUM(Amount) AS TotalPerCustomer
          FROM ORDERS
          GROUP BY CustomerID;

SQL GROUP BY Statement

The GROUP BY statement is used to group rows that have the same values into summary rows. It is often used with aggregate functions like COUNT(), SUM(), MAX(), MIN(), and AVG() to perform operations on each group of data.

Syntax:

SELECT column_name(s)
          FROM table_name
          WHERE condition
          GROUP BY column_name(s)
          ORDER BY column_name(s);

SQL CREATE VIEW Statement

In SQL, a VIEW is a virtual table that is based on the result set of a SQL query. It contains rows and columns, similar to a real table, but does not store the data itself. Instead, the data is pulled dynamically from the underlying table(s) every time the view is queried.

Syntax:

CREATE VIEW view_name AS
          SELECT column1, column2, ...
          FROM table_name
          WHERE condition;

Note: A view always reflects the latest data. When you query a view, it retrieves the current data from the underlying table(s).

SQL GRANT and REVOKE Statements

GRANT

The GRANT statement is a Data Control Language (DCL) command used to provide privileges or access rights to users on database objects such as tables, views, or procedures.

  • It grants specific permissions to users or roles.
  • Permissions must be defined for every individual user or role.
  • When access is decentralized, granting permission becomes more flexible.

Syntax:

GRANT privilege_name ON object_name TO {user_name | PUBLIC | role_name};

Example - Grant SELECT Permission

GRANT SELECT ON CUSTOMERS TO read_user;

Example - Grant SELECT and UPDATE Permissions

GRANT SELECT, UPDATE ON PRODUCTS TO analyst_role;

Example - Grant Permission to All Users

GRANT SELECT ON CUSTOMERS TO PUBLIC;

Using GRANT with a Pattern Matching View

In some cases, permissions may be granted to views that are built using pattern matching (LIKE) filters.

-- Create view for customers starting with 'A'
          CREATE VIEW Customers_Start_A AS
          SELECT CustomerName, Country
          FROM CUSTOMERS
          WHERE CustomerName LIKE 'A%';
          
          -- Grant permission to view
          GRANT SELECT ON Customers_Start_A TO support_user;

REVOKE

The REVOKE statement is also a DCL command, used to remove access rights or permissions that have previously been granted to users or roles.

  • It removes specific privileges from users or roles.
  • If a user loses access, all permissions they granted to others will also be revoked.
  • With decentralized access, revoking access can be complex.

Syntax:

REVOKE privilege_name ON object_name FROM {user_name | PUBLIC | role_name};

Example - Revoke SELECT Permission

REVOKE SELECT ON CUSTOMERS FROM read_user;

Example - Revoke SELECT and UPDATE

REVOKE SELECT, UPDATE ON PRODUCTS FROM analyst_role;

Example - Revoke Permission from All Users

REVOKE SELECT ON CUSTOMERS FROM PUBLIC;

Revoke Permission from a Pattern Matching View

-- Revoke access to the pattern-based view
          REVOKE SELECT ON Customers_Start_A FROM support_user;

SQL COMMIT and ROLLBACK Statements

SQL COMMIT

The COMMIT command is used to save all changes made in the current transaction. Once a transaction is committed, its changes are made permanent and visible to all users. After a COMMIT, you cannot undo the changes.

Syntax:

COMMIT;

Sample Data - EMPLOYEES Table

EMPID | EMP_NAME | AGE | ADDRESS         | SALARY
          ------------------------------------------------------
          101   | JOHN     | 32  | CALIFORNIA      | 5000$
          102   | DANIEL   | 38  | LOS ANGELES     | 2900$
          103   | MARIA    | 27  | TEXAS           | 4900$
          104   | JOY      | 39  | FLORIDA         | 3200$
          105   | ROOT     | 28  | SAN FRANCISCO   | 2800$
          106   | MARCUS   | 27  | CALIFORNIA      | 1000$

Example - Delete Employees Aged 27 and Commit

DELETE FROM EMPLOYEES
          WHERE AGE = 27;
          
          COMMIT;

Query the Table

SELECT * FROM EMPLOYEES;

Output

EMPID | EMP_NAME | AGE | ADDRESS         | SALARY
          ------------------------------------------------------
          101   | JOHN     | 32  | CALIFORNIA      | 5000$
          102   | DANIEL   | 38  | LOS ANGELES     | 2900$
          104   | JOY      | 39  | FLORIDA         | 3200$
          105   | ROOT     | 28  | SAN FRANCISCO   | 2800$

After executing COMMIT, the deletions are saved permanently and visible to all users.

SQL ROLLBACK

The ROLLBACK command is used to undo changes made in the current transaction. This is useful in cases where an error occurs, or you want to revert to the previous state before committing.

Syntax:

ROLLBACK;

Example - Delete Employees Aged 27 and Rollback

DELETE FROM EMPLOYEES
          WHERE AGE = 27;
          
          ROLLBACK;

Query the Table

SELECT * FROM EMPLOYEES;

Output

EMPID | EMP_NAME | AGE | ADDRESS         | SALARY
          ------------------------------------------------------
          101   | JOHN     | 32  | CALIFORNIA      | 5000$
          102   | DANIEL   | 38  | LOS ANGELES     | 2900$
          103   | MARIA    | 27  | TEXAS           | 4900$
          104   | JOY      | 39  | FLORIDA         | 3200$
          105   | ROOT     | 28  | SAN FRANCISCO   | 2800$
          106   | MARCUS   | 27  | CALIFORNIA      | 1000$

Here, ROLLBACK undoes the delete operation and restores the data to its original state.

COMMIT vs ROLLBACK

Parameter COMMIT ROLLBACK
Definition Saves the current transaction permanently Undoes changes of the current transaction
Undo possible? No Yes (until COMMIT)
When Used After successful execution On failure, abort, or cancel
Visibility Changes visible to all Changes are undone and not visible

Conclusion: Use COMMIT to make permanent changes and ROLLBACK to revert when issues arise during a transaction.
